# What this CSF `action.yaml` is for

This `action.yaml` is a **post-processing and inspection workflow** for a CSF tower model.

It does **not** define the tower geometry itself.  
Instead, it tells CSF **which checks, reports, plots, and exports must be produced** once the geometry YAML already exists.

In this case, the file is configured to work on a **HISTWIN steel tower example** and to generate a compact set of outputs for:

- geometric verification,
- section-property inspection,
- weight-law visualization,
- continuous property plots,
- integrated volume reporting,
- area breakdown by weight,
- and export tables for structural-solver workflows.

---

## Overall role of the file

The file acts as an **execution plan** for CSF actions.

It organizes the work into two parts:

- `stations`: named lists of z-coordinates where selected actions must evaluate the section;
- `actions`: the actual operations that CSF must run.

So, this file answers two practical questions:

- **where** should the tower be checked;
- **what** should CSF compute, plot, and export.

---

## Station sets defined in this file

Three station groups are declared.

### `station_ends`
```yaml
[0.0, 76.15]
```

This contains only the two boundary sections:

- `0.0` = tower base
- `76.15` = tower top

It is useful when only the end sections must be inspected.

### `station_mid`
```yaml
[38.075]
```

This is a single control section at mid-height.

It is useful for a quick spot check near the middle of the tower.

### `station_dense`
```yaml
[0.0, 10.0, 21.8, 30.0, 38.075, 48.4, 60.0, 76.15]
```

This is a denser discrete sampling along the height.

It includes:

- both ends,
- intermediate control sections,
- and the two segment-joint elevations.

It is useful when a more detailed discrete inspection is needed.

## Actions defined in this file

## 1. `plot_volume_3d`

This action produces a **3D geometric visualization** of the ruled volume generated by CSF between the two reference sections.

Purpose:

- visually check interpolation consistency,
- visually check polygon pairing,
- verify the overall shape evolution along `z`.

Main parameters used here:

- `line_percent: 60.0`  
  draws only 60% of the available generator lines to reduce clutter;

- `seed: w`  
  activates a weight-related visual appearance, so the plot is influenced by the longitudinal weight representation;

- `title: "Histwin"`  
  sets the plot title.

This is a **geometry check**, not a structural calculation.

---

## 2. `section_selected_analysis`

This action evaluates the section at selected stations and writes a **detailed section report**.

Here it uses:

```yaml
stations: station_ends
```

so only the base and top sections are reported.

Output file:

```yaml
out/histwin_properties.txt
```

Requested properties:

- `geometry`
- `A`
- `Cx`, `Cy`
- `Ix`, `Iy`, `Ixy`
- `Ip`
- `I1`, `I2`
- `rx`, `ry`
- `Wx`, `Wy`
- `J_sv_cell`
- `Q_na`
- `J_s_vroark`
- `J_s_vroark_fidelity`

Purpose:

- inspect the actual section geometry at chosen stations,
- extract the main geometric and mechanical properties,
- compare end-section behavior,
- document torsion-related estimates.

This is the main **discrete section-report action** in the file.

---

## 3. `plot_weight`

This action plots the longitudinal **polygon weights / homogenization factors** along the tower height.

Outputs:

- `stdout`
- `out/histwin_weights.jpg`

Purpose:

- inspect how weights vary along `z`,
- verify that the intended longitudinal laws are being applied,
- visually check degradation or participation trends.

This is useful when the model includes non-uniform participation through `w(z)`.

---

## 4. `plot_properties`

This action plots selected section properties as **continuous curves** along the member axis.

Outputs:

- `stdout`
- `out/histwin_properties.jpg`

Properties plotted:

- `A`
- `I1`
- `Ip`
- `J_sv_cell`

Internal sampling parameter:

```yaml
num_points: 80
```

So CSF internally evaluates 80 points between the tower base and top to build smooth curves.

Important point:

This action does **not** use the named station sets for its sampling.  
It creates its own internal continuous sampling.

Purpose:

- inspect how the main properties evolve continuously,
- detect smooth trends or local variations,
- obtain a quick visual summary of section evolution along the tower.

---

## 5. `volume`

This action computes a **volume report integrated along `z`** using Gauss-Legendre quadrature.

It uses:

```yaml
stations: station_ends
```

Outputs:

- `stdout`
- `out/histwin_volume.txt`

Parameters:

- `n_points: 10`
- `fmt_display: ".6f"`

Reported quantities include:

- occupied volume,
- homogenized volume.

Purpose:

- quantify the geometric volume,
- quantify the weight-adjusted volume,
- provide an integrated global measure along the member.

This is a **global longitudinal integration** action.

---

## 6. `section_area_by_weight`

This action reports section area grouped by weight value at selected stations.

It uses:

```yaml
stations: station_ends
```

Outputs:

- `stdout`
- `out/histwin_weight.txt`

Parameters:

- `w_tol: 0.0`
- `include_per_polygon: false`
- `fmt_display: ".6g"`

Reported quantities include:

- occupied area,
- homogenized area.

Purpose:

- show how much area belongs to each weight group,
- understand how weighted participation changes the effective section contribution,
- get a grouped summary rather than a polygon-by-polygon listing.

Because `include_per_polygon` is `false`, the output stays compact.

---

## 7. `write_sap2000_geometry`

This action exports section-property tables for **general structural-solver workflows**.

Despite the name, it is not limited conceptually to SAP2000.  
It can be useful for workflows targeting tools such as SAP2000, OpenSeesPy, or custom downstream processing.

Output file:

```yaml
out/histwin_template_pack.txt
```

It is configured with:

- `stations: station_dense`
- `n_intervals: 10`
- `material_name: "Steel S355"`
- `E_ref: 2.1e+11`
- `nu: 0.30`
- `mode: "BOTH"`
- `include_plot: false`

Purpose:

- generate a solver-oriented template pack,
- export station-based geometry/property data,
- attach reference elastic parameters for downstream use,
- prepare structured tabular data for external structural models.

Important note from this configuration:

`stations: station_dense` is explicitly provided, so the export uses that station set.  
The comment also states that `n_intervals` is not used when stations are explicitly defined.

## What this file produces in practice

Running this action file gives a compact but useful output set:

- a 3D geometry plot of the tower volume,
- a detailed end-section report,
- a plot of weight evolution,
- a plot of continuous section properties,
- an integrated volume report,
- an area-by-weight report,
- and a solver-oriented export pack.

So the file combines:

- **visual checks**,
- **local section checks**,
- **global integrated checks**,
- and **external export preparation**.

## Practical interpretation

In operational terms, this `action.yaml` is a **verification and export script** for a CSF tower model.

It is meant to help the user:

1. verify that the ruled geometry looks correct;
2. inspect the section properties at key stations;
3. inspect how weights vary along the height;
4. visualize continuous property evolution;
5. compute integrated geometric and homogenized volume;
6. understand weighted area contribution;
7. export solver-ready tabular data.
