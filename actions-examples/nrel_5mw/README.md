## 2. Tower Geometry

### 2.1 Cross-section definition

The tower shell is a **linearly tapered circular annulus**. The two boundary sections are:

| Parameter | Base - S0 (z = 0.00 m) | Head - S1 (z = 87.60 m) |
|-----------|:----------------------:|:------------------------:|
| Outer diameter D_ext | 6.000 m | 3.870 m |
| Wall thickness t | 35.10 mm | 24.70 mm |
| Outer radius R_ext | 3.0000 m | 1.9350 m |
| Inner radius R_int | 2.9649 m | 1.9103 m |

At any intermediate elevation z the geometry is obtained by **linear interpolation of R_ext and R_int** between S0 and S1:

```
R_ext(z) = R_ext,0 · (1 − z/H) + R_ext,1 · (z/H)
R_int(z) = R_int,0 · (1 − z/H) + R_int,1 · (z/H)
```

For a circular annulus this is equivalent to linear interpolation of D_ext and t independently. Both formulations produce identical intermediate geometry.

### 2.2 CSF geometry model

The geometry is encoded in a YAML file generated by `writegeometry_v6_twist.py` with the following parameters:

```
S0  (z = 0.00 m):  circle,  de = 6.000 m,  tg_base = 0.0351 m,  N = 2048
S1  (z = 87.60 m): circle,  rdx = rdy = 3.870 m,  R = 1.935 m,  tg_head = 0.0247 m,  N = 2048
twist_deg = 0
```

Setting `R = rdx/2 = rdy/2 = 1.935 m` causes the rounded-rectangle generator to degenerate into a **perfect circle**, so both boundary sections are circular annuli encoded as regular N-sided inscribed polygons.

Each section polygon is tagged `@cell` (without an explicit `@t=` token), which activates the **automatic thickness** path in CSF (see Section 3).

## 3. Torsional Constant Method - `@cell` with Automatic Thickness

### 3.1 CSF routine

The torsional constant J is computed by `compute_saint_venant_J_cell`, which implements the **Bredt-Batho closed thin-walled cell formula**:

```
J = 4 · A_mid² / (P_mid / t)
  = 4 · A_mid² · t / P_mid
```

where A_mid is the area enclosed by the wall midline and P_mid is the midline perimeter.

### 3.2 Automatic thickness (`@cell` without `@t=`)

When no explicit thickness token is present in the polygon name, CSF deduces the wall thickness from the polygon geometry at each boundary section:

```
t = 2 · A_cell / P_cell
```

where A_cell is the net cross-sectional area (outer polygon area minus inner polygon area) and P_cell is the total perimeter of both loops (outer + inner). This formula is exact for a thin-walled circular annulus with uniform thickness.

### 3.3 Thickness along the member axis

Because no explicit `@t=` token is provided, CSF does **not** interpolate a scalar thickness between the two endpoint values. Instead, it constructs the full interpolated cross-section at every queried elevation z by linearly interpolating the polygon vertices between S0 and S1, and then computes the thickness **locally** at that section using the same automatic formula:

```
t(z) = 2 · A(z) / P(z)
```

where A(z) and P(z) are the net area and total perimeter of the interpolated polygon at elevation z. This is a **continuous field function**, evaluated independently at each z - not a linear interpolation of the endpoint thicknesses t₀ and t₁.

The practical consequence is that t(z) follows the geometry of the interpolated section exactly. For a linearly tapered circular annulus, A(z) varies quadratically in z while P(z) varies linearly, so t(z) is a rational function of z - not a straight line between t₀ and t₁, as the plot of t vs z in Figure 1 confirms (the dashed blue curve is slightly nonlinear).

The thickness values extracted at the 11 verification stations are listed in Section 5.4.

### 3.4 Applicability

The Bredt-Batho formula is valid under the following assumptions, all of which are satisfied by this geometry:

- Single closed thin-walled cell
- Uniform wall thickness around the perimeter at each section
- t ≪ R (thin-wall condition: t/R ≈ 0.012 at base, 0.013 at head)

For a circular annulus the Bredt-Batho result coincides with the exact Saint-Venant torsion constant: J = π/2 · (R_ext⁴ − R_int⁴). The polygon discretisation introduces a small finite error that decreases as O(1/N²) - see Section 6.

## 6. Polygon Discretisation Error and Convergence

The CSF model approximates each circular cross-section as a regular N-sided inscribed polygon. The polygon area underestimates the exact circle area by O(1/N²):

```
ΔA/A = −π²/(3N²) + O(1/N⁴)
```

The table below shows the convergence of the maximum deviation |Δ%| across all 11 stations relative to the NREL reference values, for the quantity with the largest error (TwGJStif):

| N    | Max Δ% J vs NREL | Ratio vs previous N | Total mass Δ% |
|-----:|----------------:|--------------------:|--------------:|
|   64 |          9.49% |                  - |        +5.89% |
|  512 |          0.149% |             ~63.7× |        +0.092% |
| 1024 |          0.041% |              ~3.6× |        +0.023% |
| 2048 |          0.015% |              ~2.7× |        +0.006% |

The improvement from N=64 to N=512 is large because at N=64 the **vertex interpolation** between the two boundary polygons introduces a geometric distortion at intermediate elevations: the interpolated section is not a true circular annulus, and its area deviates significantly from the linearly tapered exact geometry. At N=512 and above this effect is suppressed and the residual error is purely the classical polygon-vs-circle discretisation error, which converges quadratically.

At N=2048 the polygon area error is −0.0006% (uniform), and all derived FAST quantities are within **0.015%** of the NREL reference values at every station.

## 9. Observations

**Geometry.** The boundary section parameters in `nrel.ps1` (D_ext, t at base and head) reproduce the NREL specification exactly. The CSF vertex-interpolation scheme is geometrically consistent with linear tapering of R_ext and R_int, which is the standard beam element assumption.

**Torsional constant.** The `@cell` tag without explicit `@t=` activates the automatic thickness path. For a closed single-cell section with uniform thickness the formula t = 2A/P is exact, and the resulting Bredt-Batho J is numerically equivalent to the polar second moment of area Ip = Ix + Iy, as expected for a circular cross-section. This coincidence is specific to circular geometry and does not hold for general closed thin-walled sections.

**Discretisation.** The quadratic convergence of the polygon approximation implies that N=2048 is adequate for all practical purposes. A further doubling to N=4096 would reduce the remaining error by approximately another factor of four, to below 0.004%, but this level of accuracy is not required for aeroelastic simulation input.

**NREL reference values.** Back-calculation from the tabulated FAST quantities (e.g. J = TwGJStif / G, A = TwEAStif / E) confirms that the NREL values are based on the exact analytical circular annulus geometry, with no polygon approximation. Agreement between the analytical solution and the tabulated values is better than 0.004% for all quantities.

## 10. The Continuous Section Field Concept

### 10.1 From discrete stations to a continuous field

The conventional approach to beam structural modelling - as used in FAST/OpenFAST and most aeroelastic codes - represents the cross-section properties as a **finite table of values at discrete stations**. For the NREL 5-MW tower this means 11 rows in Table 6-1, each describing the section at a specific elevation. Properties between stations are recovered by the solver through linear interpolation of the tabulated values.

CSF takes a fundamentally different approach. The geometry is defined by **two boundary sections** (S0 at z = 0 and S1 at z = H), and the library constructs a **continuous field function** over the entire height:

```
P(z)  for any z ∈ [z₀, z₁]
```

where P is any section property - area A, second moments Ix and Iy, torsional constant J, wall thickness t, section moduli, radii of gyration, and so on. Each property is a smooth function of the axial coordinate z, evaluated by interpolating the polygon vertices between S0 and S1 and computing the section analysis at that exact elevation.

### 10.2 Practical consequence

The plot of J_sv_cell and t(z) shown below is not a piecewise-linear curve through 11 points - it is computed on **100 equally spaced stations** along the height, each independently evaluated from the interpolated geometry. The result is a smooth, continuous curve that reflects the true variation of the cross-section properties along the member axis.

[![J_sv_cell and wall thickness t as continuous functions of elevation z](https://raw.githubusercontent.com/giovanniboscu/continuous-section-field/main/actions-examples/nrel_5mw/nrel_properties.jpeg)](https://github.com/giovanniboscu/continuous-section-field/blob/main/actions-examples/nrel_5mw/nrel_properties.jpeg)

*Figure 1. J_sv_cell (left axis, solid purple) and wall thickness t (right axis, dashed blue) as continuous functions of elevation z, evaluated at 100 stations. Values at the 11 NREL reference stations are marked. The continuity of t(z) reflects the local computation of 2A(z)/P(z) at each elevation, not a linear interpolation between endpoint values.*

For the NREL tower - a linearly tapered circular annulus - the continuous field produces:

- **A(z)**: varies as a quadratic function of z (product of two linearly varying radii)
- **Ix(z), J(z)**: vary as a quartic function of z (fourth power of linearly varying radii)
- **t(z)**: varies as a rational function of z (ratio of quadratic area to linear perimeter)

None of these are linear in z, yet all are captured exactly by the continuous field without any piecewise approximation.

### 10.3 Advantages over discrete station tables

The continuous field representation offers several practical advantages:

**Arbitrary station density.** Properties can be extracted at any number of elevations without re-running the geometry definition. The 11 NREL stations are simply 11 queries to the same continuous function. A solver requiring 100 or 1000 stations receives the same geometric fidelity without any modification to the model.

**No interpolation artefacts.** In discrete station models, the solver must interpolate between tabulated values. For quantities that vary non-linearly with z (such as J ∝ R⁴), linear interpolation between stations introduces a systematic error. The continuous field evaluates the exact geometry at every requested elevation, eliminating this source of error entirely.

**Consistent derived quantities.** When properties are extracted at any z, all quantities - A, Ix, J, t, section moduli - are mutually consistent because they are derived from the same interpolated geometry. In a discrete table, rounding or truncation of individual columns can introduce small inconsistencies between related quantities.

**Natural support for tapered and morphing sections.** The vertex-interpolation scheme extends naturally to sections that change shape along the member axis - not just scale. The twist tower example (circle at base → rounded rectangle at head, with angular twist) is handled by the same framework with no change to the analysis pipeline.

### 11.4 Relation to the FAST/OpenFAST input format

The 11-station table produced in this verification (Section 5) is one particular sampling of the continuous field at the elevations specified by NREL. It is provided for direct compatibility with the FAST ElastoDyn tower input format. The underlying CSF model, however, is not limited to these stations and can produce output at any resolution required by the downstream application.
